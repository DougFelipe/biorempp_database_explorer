Docker Compose - Guia rapido para TI (BioRemPPDBX)

Stack oficial deste projeto
- Frontend: React + Vite + TypeScript
- Backend: Node/Express
- Banco: SQLite read-only em runtime
- Servico principal: bioremppdbx
- Proxy (prod): bioremppdbx-nginx

Importante: subpath institucional
- Variavel canonica: BIOREMPP_URL_BASE_PATH
- Valor alvo institucional: /bioremppdbx/
- Sempre manter com barra inicial e final.

---

1) Pre-condicoes
- Docker e Docker Compose instalados.
- Executar comandos na raiz do projeto.
- Arquivo .env configurado (pode partir de .env.example).

Variaveis minimas esperadas:
- BIOREMPP_URL_BASE_PATH=/bioremppdbx/
- BIOREMPP_URL_BASE_PATH_NO_TRAILING=/bioremppdbx
- VITE_BIOREMPP_URL_BASE_PATH=/bioremppdbx/
- PORT=3000
- DOMAIN=localhost (ou dominio institucional)

---

2) Validar composicao antes de subir

Profile dev:
docker compose --profile dev config

Profile prod:
docker compose --profile prod config

Profile prod + ssl local:
docker compose --profile prod --profile prod-ssl-local config

---

3) Ambiente de desenvolvimento (hot reload)

Subir:
docker compose --profile dev up -d --build

Validar:
docker compose --profile dev ps

URLs esperadas:
- App: http://localhost:5173/bioremppdbx/
- API via proxy Vite: /bioremppdbx/api/*
- API direta (host): http://localhost:3101

Parar:
docker compose --profile dev down --remove-orphans

---

4) Ambiente de producao (HTTP + Nginx)

Subir limpo:
docker compose --profile prod down --remove-orphans
docker compose --profile prod up -d --build

Validar status:
docker compose --profile prod ps

Healthchecks:
- Externo (via subpath): http://localhost/bioremppdbx/api/health
- Nginx: http://localhost/health

Containers esperados:
- bioremppdbx (healthy)
- bioremppdbx-nginx (healthy)

URL esperada:
- http://localhost/bioremppdbx/

Parar:
docker compose --profile prod down --remove-orphans

---

5) Homologacao com SSL local (opcional)

Objetivo:
- Simular HTTPS local. Em producao institucional, o TLS deve terminar no ingress/proxy institucional.

Arquivos esperados:
- SSL_CERT_PATH=./.docker/certs/biorempp-local.crt
- SSL_KEY_PATH=./.docker/certs/biorempp-local.key

Subir:
docker compose --profile prod --profile prod-ssl-local up -d --build

Validar:
docker compose --profile prod --profile prod-ssl-local ps

URLs esperadas:
- HTTP mock (redireciona): http://localhost:8081
- HTTPS mock: https://localhost:8443/bioremppdbx/

Parar:
docker compose --profile prod --profile prod-ssl-local down --remove-orphans

---

6) Comandos operacionais uteis

Logs da aplicacao:
docker compose --profile prod logs bioremppdbx --tail 200

Logs do Nginx:
docker compose --profile prod logs bioremppdbx-nginx --tail 200

Rebuild completo:
docker compose --profile prod up -d --build --force-recreate

Remover orfaos:
docker compose --profile prod down --remove-orphans

---

7) Checklist de troubleshooting rapido

Se a app nao navega e fica na Home:
- Confirmar BIOREMPP_URL_BASE_PATH e VITE_BIOREMPP_URL_BASE_PATH com mesmo valor (/bioremppdbx/).
- Hard refresh no browser (cache de bundle antigo pode manter rotas quebradas).
- Conferir se /bioremppdbx/api/health responde 200.

Se logo nao aparece:
- Testar: http://localhost/bioremppdbx/BIOREMPP_LOGO.png
- Se 200, problema e cache local do frontend.

Se API falha em dev:
- Verificar VITE_DEV_API_ORIGIN (padrao http://127.0.0.1:3101).
- Conferir se container dev expos 3101.

Se houver erro de container legado:
- Rodar down com --remove-orphans para limpar servicos antigos renomeados.
